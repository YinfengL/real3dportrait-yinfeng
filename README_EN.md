# real3dportrait-cntech

This repository is an engineering adaptation of [Real3D-Portrait](https://github.com/yerfor/Real3DPortrait) (ICLR 2024 Spotlight). It keeps the original inference pipeline and adds a FastAPI HTTP service, Edge-TTS integration, long-text chunking, asynchronous task tracking, and chunk-based inference for internal digital human generation and project handoff.

The upstream Real3D-Portrait project mainly provides Gradio WebUI and CLI inference. This version focuses on service-oriented usage and deployment. It extends the original image-and-audio driven pipeline into a text-to-video generation service:

```text
Text input
→ TTS audio generation
→ audio feature extraction
→ Real3DPortrait inference
→ talking portrait video output
```

---

## Main Differences from the Upstream Project

| Aspect | Upstream Real3D-Portrait | This Project |
|---|---|---|
| Usage | Gradio WebUI / CLI | FastAPI HTTP API, while keeping the original inference entry |
| Input | Source image + driving audio / video | Text input through TTS, with audio input also supported |
| Long text | No service-level chunking in the default inference flow | Sentence splitting, chunk generation, task-level manifest |
| Task execution | Demo / script-style execution | Background task execution with status query and file download |
| Concurrency | No GPU concurrency window in the original demo flow | Sliding-window chunk inference, default window size 3 |
| Static cache | Static visual preprocessing is more tightly coupled with inference | Reuses static features for the same portrait, pose and background |
| Positioning | Official research implementation and demo | Internal service-oriented handoff version |

## Environment Setup

Please first refer to the upstream installation guide:

```text
docs/prepare_env/install_guide.md
docs/prepare_env/requirements.txt
```

The upstream project recommends Python 3.9, PyTorch 2.0.1 and CUDA 11.7. This project has also been adapted in an AutoDL environment. If you use CUDA 12.1 or another CUDA version, check the custom CUDA operator paths carefully.

Typical setup:

```bash
conda create -n real3dportrait python=3.9
conda activate real3dportrait

conda install conda-forge::ffmpeg
conda install pytorch==2.0.1 torchvision==0.15.2 torchaudio==2.0.2 pytorch-cuda=11.7 -c pytorch -c nvidia
conda install pytorch3d::pytorch3d

pip install cython
pip install openmim==0.3.9
mim install mmcv==2.1.0

pip install -r docs/prepare_env/requirements.txt -v
pip install fastapi uvicorn edge-tts
```

Check that `ffmpeg` is available:

```bash
ffmpeg -version
```

The MediaPipe segmentation model is usually downloaded automatically on first run. If the server has no internet access, download it manually:

## Start the Service

```bash
cd real3dportrait-cntech
conda activate real3dportrait
export PYTHONPATH=./

python main_api.py
```

The service listens on:

```text
0.0.0.0:8000
```

During startup, the Real3D-Portrait inference models and the TTS module are loaded. When a log similar to `Models and TTS loaded` appears, the service is ready.

## API

### POST `/generate-batch`: Start a Generation Task

```bash
curl -X POST http://localhost:8000/generate-batch \
  -H "Content-Type: application/json" \
  -d '{
    "src_image_path": "data/raw/examples/Macron.png",
    "text_input": "Hello, I am a digital human assistant.",
    "drv_pose_path": "data/raw/examples/May_5s.mp4"
  }'
```

Example response:

```json
{
  "status": "started",
  "task_id": "20240526_143022"
}
```

#### Request Parameters

| Parameter | Type | Default | Description |
|---|---|---|---|
| `src_image_path` | string | required | Source portrait image path |
| `text_input` | string | - | Text to synthesize; choose either this or `drv_audio_path` |
| `drv_audio_path` | string | - | Driving audio path, such as `.wav` or `.mp3` |
| `drv_pose_path` | string | `data/raw/examples/May_5s.mp4` | Head pose driving video |
| `bg_image_path` | string | `""` | Background image path; empty value uses the default background logic |
| `blink_mode` | string | `period` | Blink mode, such as `period` or `none` |
| `temperature` | float | `0.2` | Sampling temperature for audio-to-expression generation |
| `mouth_amp` | float | `0.45` | Mouth motion amplitude |
| `out_mode` | string | `final` | `final` for final video only; `concat_debug` for debug visualization |
| `min_face_area_percent` | float | `0.2` | Minimum face area ratio; small faces may be cropped or enlarged |

### GET `/task-status/{task_id}`: Query Task Status

Example response:

```json
{
  "task_id": "20240526_143022",
  "status": "completed",
  "generated_files": ["part_0000.mp4", "part_0001.mp4"],
  "error": null
}
```

Possible `status` values:

```text
running / completed / failed
```

### GET `/download/{task_id}/{filename}`: Download a Generated Video

```bash
curl -O http://localhost:8000/download/20240526_143022/part_0000.mp4
```

### Python Client Example

```python
import time
import requests

BASE = "http://localhost:8000"

resp = requests.post(f"{BASE}/generate-batch", json={
    "src_image_path": "data/raw/examples/Macron.png",
    "text_input": "Welcome to the digital human generation service.",
    "drv_pose_path": "data/raw/examples/May_5s.mp4",
})

task_id = resp.json()["task_id"]

while True:
    result = requests.get(f"{BASE}/task-status/{task_id}").json()
    if result["status"] in ("completed", "failed"):
        break
    time.sleep(2)

if result["status"] == "completed":
    for filename in result["generated_files"]:
        video = requests.get(f"{BASE}/download/{task_id}/{filename}")
        open(filename, "wb").write(video.content)
else:
    print(result.get("error"))
```

Task outputs are saved under:

```text

```

A task directory usually contains:


`manifest.json` records the task status, generated file list and error messages.

## FAQ

### BFM-related `.mat` or `.bin` files are missing

If the missing files are:

```text
01_MorphableModel.mat
BFM_exp_idx.mat
BFM_front_idx.mat
BFM_model_front.mat
Exp_Pca.bin
facemodel_info.mat
```

the large BFM files have not been prepared. Please follow the model download section.

If the missing files are:

```text
select_vertex_id.mat
similarity_Lm3D_all.mat
```

pull the latest repository first, because these two small metadata files are kept in Git.

### HuBERT model path error

Check the `hubert_model_dir` setting in:

```text
data_gen/utils/process_audio/extract_hubert.py
```

If the server cannot access HuggingFace, manually download `facebook/hubert-large-ls960-ft` and point `hubert_model_dir` to the local path.

### Custom CUDA operators fail to compile

This project has been adapted in an environment using CUDA 12.1 paths. If your CUDA version is different, check:

```text
modules/eg3ds/torch_utils/custom_ops.py
```

and update `extra_include_paths` and `extra_ldflags` to match the CUDA path on your machine.

### Edge-TTS network error

Edge-TTS depends on Microsoft online speech services. Make sure the server can access the internet, or replace it with an internal TTS service.

## Handoff Checklist

Before handing over the project, check the following items:

1. The GitHub repository is up to date.
2. `checkpoints/` has been prepared outside Git.
3. Large files in `deep_3drecon/BFM/` have been prepared.
4. The Conda environment can import project modules.
5. `ffmpeg` is available.
6. The original `inference/real3d_infer.py` path runs successfully.
7. `main_api.py` starts successfully.
8. A short text request can generate at least one video chunk.
9. Output files appear under `infer_out/batch_tasks/`.
10. The next developer has access to the private repository and the external model files.
